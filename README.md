# Auto Equation Numbering for Obsidian

本插件为 Obsidian 提供显示公式（Display Math, `$$ ... $$`）的自动编号与正文交叉引用同步功能。旨在帮助学术研究人员、教师及学生在撰写包含大量数学公式的笔记时，实现公式标签的高效管理。

---

## 核心功能

### 1. 显示公式自动编号
- **自动解析与标记**：解析 Markdown 文档中的所有独立显示公式（`$$ ... $$`），并在公式末尾追加符合 LaTeX 规范的 `\tag{n}`。
- **免干扰设计**：不对行内公式（`$...$`）和 Markdown 代码块（如 ` ``` ` 块内代码）进行任何处理，保证源码安全性。

### 2. 正文交叉引用同步
支持 LaTeX 风格的公式标签声明与正文内的链接引用：
- **声明标签**：在显示公式内部使用 `\label{label_name}` 声明标识符。
- **引用格式**：在正文中使用 Markdown 链接指向该标识符，即 `[显示文本](#label_name)`（例如 `[式（）](#eq:model)`）。
- **动态填充与同步**：
  - 支持空括号占位符（如 `[式（）](#label_name)`），在首次执行更新时，插件会自动将当前公式编号填入括号内。
  - 当由于插入新公式或调整段落顺序导致公式编号发生变更时，重新运行更新命令即可使正文中的全部引用链接同步更新，避免手动维护的错误率。

### 3. 多样化编号格式定制
支持全局或单篇笔记独立的公式编号样式定制，满足不同期刊或学术规范的排版要求：
- **Default (默认)**：`1`、`2`、`3`...
- **A-\***：前缀样式 `A-1`、`A-2`...
- **2.\***：小节点样式 `2.1`·`2.2`...
- **2-\***：带括号样式 `(2-1)`、`(2-2)`...
- **Custom (自定义)**：支持自定义输入，其中 `*` 占位符将被替换为公式的顺序序号。

### 4. 重复公式去重机制
- 插件在分析公式时会提取并标准化其 LaTeX 核心结构（剔除无意义的空白字符与已有 tag）。
- 如果检测到同一篇笔记中存在多个完全相同的公式，默认会复用相同的公式编号以保证推导的一致性。
- 该机制可在插件设置中关闭，关闭后将为所有公式分配独立递增的编号。

### 5. 交互与控制
- **侧边控制面板**：点击功能区图标或运行 `Open Equation Numbering control panel (Sidebar)` 命令可打开侧边栏，支持针对当前笔记启用/禁用编号、切换格式及手动触发更新。
- **状态栏快捷切换**：窗口右下角提供 `🔢` 状态指示与快捷开关，便于快速切换当前笔记的自动编号状态。

## 效果演示

### 公式自动添加 Tag

**原始文本：**
```markdown
$$
\hat{\boldsymbol{y}}_c = \boldsymbol{X}_c\hat{\boldsymbol{\beta}}
$$
```

**更新后文本：**
```markdown
$$
\hat{\boldsymbol{y}}_c = \boldsymbol{X}_c\hat{\boldsymbol{\beta}}
\tag{1}
$$
```

### 交叉引用同步

**编辑中的笔记内容：**
```markdown
$$
y_i = \beta_0 + \sum_{j=1}^p \beta_j x_{ij} + \varepsilon_i
\label{eq:mlr}
$$

如[式（）](#eq:mlr)所示，我们建立了多元线性回归模型。
```

**手动触发更新（点击侧边栏 Update 或执行更新命令）后：**
1. 正文中的占位链接被自动填充为：`[式（1）](#eq:mlr)`。
2. 若在当前公式前插入其他公式导致其编号变为 `2`，再次触发更新后，引用链接将自动同步更新为：`[式（2）](#eq:mlr)`。

## 安装与构建指南

### 1. 手动安装
1. 克隆或下载本仓库至本地。
2. 在项目根目录下执行依赖安装与打包：
   ```bash
   npm install
   npm run build
   ```
3. 在您的 Obsidian 库中定位到 `.obsidian/plugins/` 目录，并创建一个名为 `obsidian-auto-equation-numbering` 的新文件夹。
4. 将编译生成的 `main.js` 以及项目中的 `manifest.json` 复制到该文件夹中。
5. 在 Obsidian 的“社区插件”设置中重新加载并启用本插件。

### 2. 开发命令
本地开发调试时，可在本目录执行以下命令监听文件变化并自动重新编译：
```bash
npm run dev
```

## 技术注意事项

- **手工 Tag 覆盖**：插件会接管显示公式中的 `\tag{...}`。若文档中存在手动写入的 `\tag`，在执行更新时会被插件的自动编号逻辑覆盖。
- **MathJax 渲染兼容处理**：由于 Obsidian 原生的 MathJax 渲染可能对 `\label{...}` 抛出解析异常，插件在重写公式时会为其自动添加 `%` 前缀（即重写为 `% \label{...}`）。该处理仅在渲染层面避免报错，不影响插件对标识符的提取与交叉引用更新。
